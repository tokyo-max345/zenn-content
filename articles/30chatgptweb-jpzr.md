# 【AI過信】「このエラー直して」と頼み続けたら、気づけば無限ループにハマり時間と心が溶けた話

正直に言います。僕はAIを完全にナメていました。

「AIを使えば、プログラミングなんてチョロい」

プログラミングを始めたばかりの僕は、そう信じて疑いませんでした。だって、ChatGPTとかGeminiとか、すごいじゃないですか。何でも答えてくれるし、コードも書いてくれる。これさえあれば、サクッと開発して、俺もAI時代の寵児に！ なんて、夢見てたんです。今思えば、本当に甘かった。プログラミング未経験で、開発の「か」の字も知らないくせに、AIを過信しすぎた結果、僕はとんでもない泥沼にハマることになります。

この記事は、そんな僕の痛い失敗談を赤裸々に告白するものです。「AIでサクッと開発」に夢見がちなあなたに、僕と同じ轍を踏んでほしくない。そんな思いで書きました。

## 「AI、マジ神！」初めての成功体験が僕を沼に引きずり込んだ

僕がプログラミングを始めたのは、ほんの半年前。簡単なWebサイトを作ってみようと一念発起したのがきっかけでした。HTMLとCSSを少し触った程度で、JavaScriptなんてチンプンカンプン。でも、AIがあれば大丈夫！と信じていました。

最初にAIに頼んだのは、Webサイトに簡単なアニメーションを追加するコードでした。

```text
Webサイトの要素がスクロールしたらふわっと表示されるJavaScriptのコードを書いてください。
```

こんなザックリしたプロンプト（AIへの指示文）なのに、AIはものの数秒でコードを生成してくれました。

```javascript
document.addEventListener('DOMContentLoaded', function() {
  const targets = document.querySelectorAll('.fade-in');

  const observer = new IntersectionObserver((entries, observer) => {
    entries.forEach(entry => {
      if (entry.isIntersecting) {
        entry.target.classList.add('is-show');
        observer.unobserve(entry.target);
      }
    });
  });

  targets.forEach(target => {
    observer.observe(target);
  });
});
```

これを自分のHTMLに貼り付けて、CSSを少し調整したら、本当に動いたんです！ 要素がスクロールに合わせて、ふわっと現れる！

「おおおおお！AI、マジ神じゃん！！！」

あの時の感動は忘れられません。プログラミングって、こんなに簡単なのか！ と。AIを使えば、俺はどんなものでも作れるんじゃないか？ そんな全能感に酔いしれていました。これが、僕がAIの沼に引きずり込まれる、最初の甘い誘惑だったんです。

:::message
AIにコードを書いてもらう時、簡単なものだと本当に一瞬で動いて感動しますよね。僕も最初の成功体験で「AIはなんでもできる」と過信しちゃいました。
:::

## 悪夢の始まり。「直して」の無限ループに突入

順調にAIと二人三脚で開発を進めていた僕ですが、ある日、悪夢のような出来事が起こりました。

Webサイトに、ユーザーが入力した内容をローカルストレージ（ブラウザにデータを保存する機能）に保存して表示する機能を実装しようとしていた時のことです。AIにコードを生成してもらい、いざ実行！ と思ったら、コンソールに赤文字のエラーがずらり。

```text
Uncaught TypeError: Cannot read properties of null (reading 'value')
```

「うわ、エラーだ…まあ、AIに聞けば一発でしょ！」

そう高を括っていた僕は、このエラーメッセージをそのままAIにコピペして、プロンプトを打ち込みました。

```text
このエラーを直してください。
Uncaught TypeError: Cannot read properties of null (reading 'value')
```

AIはすぐに修正案を提示してくれました。「要素がまだ読み込まれていない可能性があります」とのこと。言われた通りにコードを修正して、もう一度実行。

しかし、また別のエラーが！

```text
Uncaught TypeError: Cannot set properties of null (setting 'textContent')
```

「え、なんで？ さっき直したばっかりなのに…」

僕はまた、この新しいエラーメッセージをAIにコピペし、「これを直してください」と送りました。AIはまた別の修正案を提示。僕はそれを信じてコードに反映。

そして、またエラー。

この繰り返しです。AIが提案する修正を適用すると、別の箇所でエラーが発生する。そのエラーをAIに投げると、また別のエラーが発生する。まるでモグラ叩きのように、次から次へとエラーが湧いてくるんです。

:::warning alert
エラーメッセージをAIに投げ続けるのは、泥沼の入り口です。なぜそのエラーが出ているのか、AIが提案する修正が何をしているのかを理解しないままでは、永遠にループから抜け出せません。
:::

## 時間と精神がゴリゴリ削られていく感覚

「直して」の無限ループにハマってから、僕は完全に冷静さを失っていました。深夜2時。部屋の電気をつけっぱなしで、目と頭はガンガン。コーヒーを何杯も飲んで、眠気と戦いながらAIと対話していました。

「これで直るはず…頼む…！」

AIが新しいコードを提示するたびに、かすかな希望を抱き、それをコピペしては実行ボタンを押す。そして、また赤いエラーメッセージが表示されるたびに、絶望と疲労が押し寄せます。

「なんでだよ…なんで動かないんだよ…」

エラーが解決しないのは、AIのせいじゃない。僕のプロンプトが悪いのか？ もっと詳しく状況を説明しないとダメなのか？ でも、何が悪いのかも分からない。

結局、その日は朝方まで格闘しましたが、エラーは解決せず。時間だけが溶けていき、精神はボロボロになっていました。PCの画面に映る無数のエラーメッセージは、僕の心を蝕むモンスターに見えました。あの時の無力感と絶望感は、今でも鮮明に覚えています。

週末も、そのエラーに取り憑かれたように、AIとの問答を繰り返しました。まるで、AIに人生を支配されているかのような感覚。最初は楽しかった開発が、いつの間にか苦痛でしかなくなっていました。

最初は丁寧だったプロンプトも、だんだん投げやりになっていきました。

```text
なんかまた動かない。これ直して。
```

```text

```

```text
AIが言った通りにやったのに、またエラー出たんだけど。なんで？
```

```text
これもう無理。全部のコード見直して、どこがおかしいか教えて。
```

今見ると、本当にひどいプロンプトですね…。これではAIも困惑するでしょう。
:::

## なぜ僕はAIの無限ループから抜け出せなかったのか？

後になって冷静に振り返ると、僕がAIの無限ループから抜け出せなかった理由は明白でした。

### 1. エラーの根本原因を理解しようとしなかった

僕がAIにエラーを投げ続けるたびに、AIは毎回異なる修正案を提示してきました。しかし、僕はそれらの修正案がなぜ必要なのか、何を変えようとしているのかを全く理解していませんでした。

例えば、最初の `Cannot read properties of null (reading 'value')` というエラー。これは「`null`（何もない状態）のオブジェクトから`value`というプロパティを読み取ろうとした」という意味です。つまり、`value`プロパティを持った要素（例えばテキスト入力欄）が、コードが実行される時点で見つかっていないか、存在していないことが原因だったんです。

僕のコードでは、JavaScriptがHTMLの要素を読み込む前に、その要素にアクセスしようとしていました。AIは `DOMContentLoaded` イベントリスナーを追加したり、`if (element)` で要素の存在チェックを入れたりする修正案を提示してくれましたが、僕はそれが「要素がまだ読み込まれていない」という根本原因に対する対処だとは理解していませんでした。

### 2. AIの提案を鵜呑みにし続けた

AIはあくまで「統計的に最も適切な答え」を提示しているだけです。僕のコード全体の文脈や、僕が何をしたいのかを完全に理解しているわけではありません。特に、既存の複雑なコードに対してAIが修正を加える場合、その修正が他の箇所に予期せぬ影響を与える可能性も十分にあります。

僕はAIの提案を「絶対の正解」だと信じ込み、そのままコピペし続けていました。その結果、一つのエラーが直っても、AIが加えた修正が原因で、別の箇所に新しいエラーが生まれるという負の連鎖に陥ってしまったんです。

### 3. 自分の知識不足が無限ループを加速させた

これが一番大きかったと思います。プログラミングの基礎知識がなさすぎました。

*   JavaScriptの実行順序
*   DOM（Document Object Model）の操作の基本
*   エラーメッセージの読み解き方
*   デバッグの基本的な考え方

これらの基礎が全くない状態で、AIという強力なツールを扱おうとしたのが、すべての間違いの始まりでした。基礎があれば、AIが提示した修正案を自分で検証したり、エラーメッセージから根本原因を推測したりできたはずです。しかし当時の僕には、それが全くできませんでした。

AIは万能の解決策ではなく、あくまで**ツール**です。そのツールを使いこなすためには、それを使う側の人間にも一定の知識と理解が必要なんだと、この痛い経験を通じて痛感しました。
