# 【地獄】AIに「このエラー直して」と言い続けたら、沼にハマって半日溶けた話。

正直に言います。最初は完全にナメてました。

「AIってすごいじゃん！エラーなんてチョチョイのチョイで直してくれるんでしょ？」

個人開発を始めて間もない僕は、そんな根拠のない自信と甘い期待を胸に抱いていました。当時、初めて挑戦したWebアプリ開発で、ちょっとした認証機能の実装に悪戦苦闘していたんです。

何度かコードを書いては動かず、ブラウザには見慣れない赤い文字の羅列。

```text
Uncaught TypeError: Cannot read properties of undefined (reading 'email')
```
:filename=console.log

「うわ、出たよエラー。でも大丈夫、AIがいるもんね！」

そう思い、僕は得意げにChat-GPT（当時はまだGPT-3.5でしたね）にエラーメッセージと、怪しいと感じたJavaScriptのコードをコピペして投げました。これが地獄の始まりとも知らずに…。

## 「よし、これで完璧！」と思ったのに、新たなエラー爆誕

AIに初めてエラー修正を依頼した時、僕はこんなプロンプトを送りました。

```text
このJavaScriptコードでTypeErrorが発生しました。
ユーザーのemailを読み込もうとしている部分でundefinedになっているようです。
どこを修正すればいいか教えてください。

TypeError: Cannot read properties of undefined (reading 'email')

const user = response.data.user;
console.log(user.email);
```
:filename=プロンプト

すると、AIはあっという間に「おそらく`response.data`がundefinedになっている可能性があります」と指摘し、こんな修正案を提示してくれました。

```javascript
// AIからの修正案
if (response && response.data && response.data.user) {
    const user = response.data.user;
    console.log(user.email);
} else {
    console.error("ユーザーデータが見つかりません。");
}
```
:filename=AIの修正案

「おお、なるほど！ヌルチェックね！賢い！」

僕は深く考えることもなく、言われるがままにコードを修正し、意気揚々とブラウザをリリロードしました。これで完璧、今日の作業は終わりだ！そう思っていたんです。

…が、甘かった。

ブラウザのコンソールには、またしても赤い文字。しかし、さっきとは少し違うエラーメッセージが表示されていました。

```text
ReferenceError: response is not defined
```
:filename=console.log

「え、なんで？さっきまで`response`は定義されてたじゃん！？」

僕の頭の中は疑問符でいっぱいになりました。AIの修正で、かえって別のエラーを生み出してしまったのです。

:::message alert
AIにコード修正を依頼する際は、必ず修正前のコード全体や関連するコードも提示するようにしましょう。部分的なコードだけでは、AIも状況を正しく把握できず、かえって問題を引き起こすことがあります。僕のこの失敗談がまさにそれです！
:::

## 「え、なんで？」無限ループにハマった泥沼の半日

そこからの僕は、まさに泥沼でした。

「`response`が定義されてないって言われたんで、どうしたらいいですか？」
「では、`response`がどこで取得されているか確認してください。もし非同期処理なら`await`を使っていますか？」

AIの指示通りにコードを探し、`await`を追加。
↓
また別のエラー（「`await`は`async`関数内でしか使えません」）
↓
AIに相談し、関数に`async`を追加。
↓
また別のエラー（「`fetch`が定義されていません」）
↓
AIに相談し、`fetch`をインポート。
↓
また別のエラー（「クロスオリジンエラー」）

……こんなやり取りを、僕は半日近く続けていました。時計を見ると、朝の9時から始めた作業が、もう午後2時を回っています。

正直、もう何がなんだか分かりません。頭の中はエラーメッセージとAIの回答でぐちゃぐちゃ。

```text
俺: TypeError: Cannot read properties of undefined (reading 'email') と出ます。

AI: response.data.user が undefined の可能性があるので、ヌルチェックを追加してください。
```
:filename=AIとのやり取り1

```javascript
// 修正後
if (response && response.data && response.data.user) {
    const user = response.data.user;
    console.log(user.email);
} else {
    console.error("ユーザーデータが見つかりません。");
}
```
:filename=AIの修正案

```text
俺: ReferenceError: response is not defined と出ました。responseがどこで定義されているか分かりません。

AI: responseはAPIからのレスポンスを受け取る変数ですので、API呼び出しの部分を確認してください。
おそらく、以下のように非同期でデータを取得しているはずです。
```
:filename=AIとのやり取り2

```javascript
// AIの提案
async function getUserData() {
    try {
        const response = await fetch('/api/user'); // 仮のAPIエンドポイント
        const data = await response.json();
        if (data && data.user) {
            const user = data.user;
            console.log(user.email);
        } else {
            console.error("ユーザーデータが見つかりません。");
        }
    } catch (error) {
        console.error("データの取得中にエラーが発生しました:", error);
    }
}
```
:filename=AIの修正案

僕は自分の書いたコードとAIの提案を交互に見比べながら、「うーん…どこが違うんだ…？」と唸るばかり。
結局、元のコードは`fetch`ではなく`axios`を使っていたり、そもそも`response`オブジェクトの構造がAIの想定とは全然違ったり…と、根本的な部分で認識のズレがあったんです。

AIは僕の「部分的な情報」から「一般的な解決策」を提示しているだけ。僕が提示する情報が不正確だと、AIも的外れな回答しかできないんですよね。

:::message
AIにエラー修正を依頼する際は、コードだけでなく、エラーが発生している状況（どのファイルで、どの関数内で、どんな処理を行っている時か）も具体的に伝えるようにしましょう。僕はそれを怠ったせいで、無限ループにハマりました。
:::

## 「もしかして、原因は俺？」ようやく気づいた痛い真実

半日溶かして、カフェインと絶望で頭がパンクしそうになった時、ふと我に返りました。

「あれ…？これって、もしかして俺がエラーメッセージをちゃんと読んでないだけ…？」

そう、AIは親切にエラーの原因になりそうな箇所や修正方法を教えてくれる。でも、それはあくまで「一般的な」推測に過ぎません。僕自身のコードの全体像や、使っているライブラリの特性、APIのレスポンス構造なんかは、AIは完璧には知りません。

僕がすべきだったのは、AIに丸投げする前に、まず**エラーメッセージを自分で読み解くこと**でした。

- `TypeError: Cannot read properties of undefined (reading 'email')`
  -> `email`を読もうとしたけど、その親オブジェクトが`undefined`だったんだな。じゃあ、`user`オブジェクトの中身はどうなってる？
- `ReferenceError: response is not defined`
  -> `response`という変数が、このコードのどこにも定義されてないってことか。じゃあ、そもそも`response`はどこから来るべき変数なんだっけ？

当たり前のことなんですけど、当時の僕は「AIが何とかしてくれる」という思考停止状態に陥っていました。エラーメッセージをググる手間すら惜しんで、すぐにAIに頼ってしまう。その結果、根本原因を特定できないまま、場当たり的な修正を繰り返してしまったんです。

この時、僕は痛いほど実感しました。AIはあくまで補助ツールであり、最終的な判断やエラーの根本原因の特定は、**人間である自分が行うべきだ**と。
