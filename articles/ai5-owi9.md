# AIに丸投げ開発で爆死した僕が学んだ、地雷を踏まないための5つの鉄則

正直に言います。最初はAIを完全にナメてました。

「AIに任せれば楽勝！」

そう信じて、個人開発で何度痛い目を見たことか。深夜まで張り付いて、結局動かないコードと睨めっこ。あの時の絶望感、今でも鮮明に覚えてます。

「ちょっとしたツールなら、AIにプロンプト投げるだけでサクッと作れるでしょ？」

軽い気持ちで始めた個人開発は、いつも爆死エンド。なんでAIはこんなにポンコツなんだ！とAIのせいにしたり、自分の実力不足に絶望したり……。そんな僕が、血と汗と涙、そして大量のコーヒーを消費しながら学んだ『AI開発で地雷を踏まないための5つの鉄則』を、今回は赤裸々に語りたいと思います。

AIにコード生成や開発を任せてみたいけど、まだ何もやったことがないプログラミング初心者の方。僕みたいに爆死しないように、ぜひこの記事を読んでいってください。

## 【地雷その1】「完璧な指示」なんて幻想だった件

僕が最初にAI開発でコケたのは、まさにここでした。

「なんかいい感じのWebアプリ作って！」

こんな漠然とした指示で、AIが僕の頭の中にある”理想のアプリ”を生成してくれると本気で思ってたんです。今考えたらアホかと。AIはエスパーじゃないんですよ。僕らの思考を読み取って、完璧なコードを吐き出してくれるなんて、そんなSFみたいな話はまだ先です。

最初の頃は、AIが返してきたコードを見て「なんだこれ、全然違うじゃん！」とイライラしてました。でも、よくよく考えたら、指示を出してる僕自身が「何を作りたいのか」を明確に言語化できてなかったんですよね。

:::message
**僕の失敗談:**
「ユーザーが投稿できる掲示板サイトが作りたい」とだけ伝えたら、AIはただのHTMLファイルと最低限のCSSを返してきた。データベース連携とかユーザー認証とか、僕が考えてた機能は一切含まれてなかった……。
:::

AIに期待通りのコードを生成してもらうためには、具体的な指示が不可欠です。例えば、掲示板サイトならこんな感じ。

- **目的:** ユーザーが自由にテキストを投稿し、他のユーザーが閲覧できる掲示板サイト
- **技術スタック:** フロントエンドはReact、バックエンドはNode.js（Express）、データベースはMongoDB
- **機能要件:**
    - ユーザー登録・ログイン機能
    - 投稿作成（タイトル、本文）
    - 投稿一覧表示（新しい順、ページネーションあり）
    - 投稿詳細表示
    - 投稿編集・削除（投稿者のみ）
- **UI/UX:** シンプルなデザインで、レスポンシブ対応

ここまで具体的に指示を出すと、AIもかなり精度の高いコードを生成してくれます。もちろん、一発で完璧なものは出ませんが、叩き台としては十分。

「AIは対話型インターフェース」ってよく言われますけど、まさにその通り。人間同士の会話と同じで、曖昧な言葉では相手に伝わりません。具体的な言葉で、丁寧に意図を伝えることが、AI開発の第一歩だと痛感しました。

## 【地雷その2】生成コードを「理解せずコピペ」は自爆行為

AIが生成したコードって、なんかそれっぽいし、とりあえず動けば「やったー！」ってなりますよね。僕もそうでした。

「お、エラー出ないじゃん！天才！」

なんて浮かれて、生成されたコードをそのままコピペして使ってたら、後でとんでもないデバッグ地獄に突き落とされました。

例えば、AIが生成したReactのコンポーネントをそのまま使ってたら、特定の条件下で無限ループに陥ってブラウザが固まる、とか。最初はどこが悪いのか全く分からず、半日くらい唸ってました。

結局、原因はAIが提案した `useEffect` の依存配列（`dependencies array`）が不適切だったから。僕がそのコードの意味をちゃんと理解せずにコピペしたのが原因です。

:::message alert
**絶対NG！**
AIが生成したコードを、意味も分からずそのままプロダクトに組み込むのは、時限爆弾を抱えるようなものです。後で必ず破綻します。
:::

じゃあどうすればいいのか？

簡単なコードレビューを自分でやる、これに尽きます。別に全てを完璧に理解する必要はないんです。

- **処理の流れを追う:** この関数は何を受け取って、何を返すのか。どんな処理をしているのか。
- **変数の役割を確認する:** この変数には何が入ってて、どこで使われているのか。
- **知らない構文や関数は調べる:** AIが使った見慣れないメソッドやライブラリがあったら、軽くググってみる。

これだけでも全然違います。もしAIが生成したコードで、全く意味が分からない部分があったら、正直にAIに質問してみるのも手です。

「この `useCallback` って、何のために使ってるの？具体的にどういう効果があるの？」

AIは意外と丁寧に答えてくれます。これを繰り返すことで、自分のプログラミングスキルも上がっていくし、AIの生成コードの品質チェックもできるようになります。まさに一石二鳥。

## 【地雷その3】AIは万能じゃない！得意・不得意の見極め方

AIに何でもかんでも任せようとして、かえって時間がかかったり、品質が落ちたりした経験、数え切れません。

「この複雑な認証システム、全部AIに設計から実装までやらせちゃお！」

そう思って、ひたすらプロンプトを練ってAIと対話した結果、出てきたのはセキュリティ的に穴だらけのコードだったり、非効率な処理がてんこ盛りだったり。結局、自分でイチから書き直した方が早かった、なんてことがしょっちゅうありました。

AIには得意なことと不得意なことがあります。これを理解して使い分けるのが、AI開発を成功させる鍵です。

**AIが得意なこと:**
- **定型作業:** ボイラープレートコード（お決まりの記述）の生成、簡単な関数の作成、テストコードの骨子作成
- **アイデア出し:** 機能の提案、デザインパターンの選択肢、エラーメッセージの文案
- **既存コードの改善提案:** リファクタリング案、パフォーマンス改善のヒント
- **エラーの原因究明のヒント:** エラースタックトレースを渡すと、原因の候補を挙げてくれる

**AIが不得意なこと:**
- **複雑なシステム設計:** 全体像を把握した上でのアーキテクチャ設計、大規模なデータベース設計
- **デバッグ:** 特定の環境でしか再現しないような複雑なバグの特定と修正
- **セキュリティ要件を満たすコード生成:** ゼロトラストとか、最新の脆弱性対策とか
- **創造的な問題解決:** 誰も試したことのないような、革新的なアルゴリズムの考案

僕の場合、最近はAIに「シンプルなCRUD（Create, Read, Update, Delete）操作のAPIエンドポイント作って」とか、「Reactのカスタムフックでこういう機能のやつ作って」みたいな、**局所的で明確なタスク**をお願いすることが多いです。全体の設計や、セキュリティが絡むような部分は、自分でじっくり考えて実装するようにしてます。

:::details 僕がAIによく頼む定型作業のプロンプト例（Node.js + Express + MongoDB）
```text
Node.jsとExpressを使って、MongoDBのUserコレクションに対する基本的なCRUD操作（作成、読み取り、更新、削除）を行うAPIエンドポイントを実装してください。
以下の要件を満たしてください。

- POST /api/users で新しいユーザーを作成
- GET /api/users で全ユーザーを取得
- GET /api/users/:id で特定のユーザーを取得
- PUT /api/users/:id で特定のユーザーを更新
- DELETE /api/users/:id で特定のユーザーを削除
- 各エンドポイントでエラーハンドリングを適切に行う
- Mongooseスキーマの定義もお願いします（name: String, email: String, password: String）
```
:::

こういうタスクはAIが本当に得意で、一瞬でかなり使えるコードを生成してくれるんですよね。まさに「相棒」って感じ。

## 【地雷その4】「AI任せ」は思考停止への入り口だった

AIに頼りきりになった時期、正直ありました。

「どうせAIがやってくれるし、自分で考える必要なくね？」

そう思って、コードの意味を深く考えず、ひたすらコピペとプロンプト調整に時間を費やしていたら、ある日突然、自分のプログラミングスキルが伸び悩んでいることに気づきました。

新しい技術を学ぶ意欲も薄れ、ちょっと複雑な問題に直面すると「AIに聞けばいいや」で思考停止。これ、本当に危険です。AIはあくまでツールであって、あなたの脳みその代わりにはなりません。

僕が経験した具体的な例だと、Reactのコンポーネントのライフサイクルについて、AIがコードを生成してくれた時に「なんでここで `useEffect` を使う必要があるんだろう？」と疑問にすら思わなかったんです。そのままコピペして、なんとなく動けばOK、と。

でも、ある時、自分でゼロからアプリケーションを設計・実装しないといけない状況になって、全然手が動かないことに愕然としました。基本的な設計思想や、各技術の深い部分が抜け落ちてたんです。

AIを使いこなしつつ、自分のスキルも向上させるためのバランス、これは本当に重要です。僕が実践しているのは、こんなことです。

1.  **AIが生成したコードの「なぜ」を考える:** なんでこの関数を使ってるの？なんでこのデータ構造なの？
2.  **自分でゼロから書いてみる:** AIに頼らず、簡単な機能でも良いので、一度自分の手で書いてみる。
3.  **AIを「学習のパートナー」にする:** わからない概念やエラーがあったら、AIに質問して解説してもらう。ただ答えを求めるだけでなく、理解するまで対話する。

:::message
**個人的な感覚:**
AIが生成したコードを理解しようと努力するプロセスこそが、最高の学習機会だと最近は思ってます。自分でググるより、AIに質問した方が、よりパーソナライズされた回答が得られるし、深掘りもしやすい。
:::

AIを「思考停止ツール」にするか、「最強の学習パートナー」にするかは、あなた次第です。僕は後者を目指すことにしました。
