# AIに丸投げしたら、僕の個人開発DBが消えた話。プログラミング初心者の『どん底体験記』

## きっかけは些細なことだった。AIで夢のアプリ開発へ

僕がAIブームに乗っかって、プログラミングの世界に足を踏み入れたのは、本当に些細なきっかけだったんです。ある日、SNSを見ていたら、「プログラミング未経験でもAIを使えば簡単にアプリが作れる！」みたいな記事が流れてきて、正直「マジかよ」って思いました。でも、心のどこかで「自分にもできるかも」ってワクワクしたんですよね。

昔から、「こんなアプリがあったら便利なのに」ってアイデアだけはたくさんあったんです。例えば、読んだ本の感想をサッと記録できるアプリとか、自分だけの秘密のレシピ帳とか。でも、プログラミングってなんか難しそうだし、専門知識がないと無理だと思って諦めてました。

それが、AIの登場で「もしかしたら、僕でも夢のアプリを作れるかもしれない！」って希望が見えてきたんです。最初は、ChatGPT（チャットジーピーティー）みたいな対話型AIに、「簡単なウェブサイトの作り方を教えて」とか、「Python（パイソン）で数字を足し算するコードを書いて」とか、本当に初歩的なことから聞いてました。AIの返答はいつも丁寧で、すぐにコードを生成してくれる。まるで、優秀な家庭教師が横にいるみたいで、もうすっかりAIに夢中になっていました。

「よし、この勢いで、前から作りたかった『読書記録アプリ』を作ってみよう！」そう心に決めたんです。目標は、本のタイトルと感想、読了日を記録して、後で検索できるようにする、ごくシンプルなもの。これならAIの力を借りれば、きっとできるはずだ！って、根拠のない自信に満ち溢れていました。

## 「これならいける！」AIが教えてくれた魔法のコード

僕がアプリ開発を始めたのは、仕事が終わってからの夜中が多かったですね。静かな部屋で、PCに向かい、AIと対話する時間が、日課になっていました。

まずは、データの保存先が必要だとAIに教えてもらい、SQLite（エスキュエルライト）というデータベースを使うことにしました。理由は、AIが「手軽に使えるよ」って教えてくれたからです。もちろん、SQLiteが何なのか、RDB（リレーショナルデータベース）って何？みたいなことは、その時は全然わかってませんでした。ただ、AIが「これを使えばいい」って言うから、信じていたんです。

AIに「PythonでSQLiteデータベースを作って、テーブルを定義するコードを書いて」とプロンプト（AIへの指示文）を送ると、あっという間にコードが返ってきました。

```python:create_db.py
import sqlite3

# データベースファイルを作成（または接続）
conn = sqlite3.connect('my_book_log.db')
cursor = conn.cursor()

# 読書記録テーブルを作成
cursor.execute('''
    CREATE TABLE IF NOT EXISTS books (
        id INTEGER PRIMARY KEY AUTOINCREMENT,
        title TEXT NOT NULL,
        author TEXT,
        感想 TEXT,
        read_date TEXT
    )
''')

# 変更をコミット
conn.commit()
# 接続を閉じる
conn.close()

print("データベースとテーブルが作成されました！")
```

「おお！なんかそれっぽい！」って、興奮しましたね。言われた通りにこのコードを実行すると、「データベースとテーブルが作成されました！」ってメッセージが表示されて、「これならいける！」と確信しました。まるで魔法みたいに、何もないところからデータベースができたんですから。

次に、このデータベースにデータを追加するコードもAIに教えてもらいました。

```python:insert_data.py
import sqlite3

conn = sqlite3.connect('my_book_log.db')
cursor = conn.cursor()

# データの挿入
cursor.execute("INSERT INTO books (title, author, 感想, read_date) VALUES (?, ?, ?, ?)",
               ('AIとの共創', 'AI太郎', 'AIの可能性を感じた一冊。', '2024-07-25'))
cursor.execute("INSERT INTO books (title, author, 感想, read_date) VALUES (?, ?, ?, ?)",
               ('Python入門', 'プログラミング次郎', '基本が学べてよかった。', '2024-07-20'))

conn.commit()
conn.close()

print("データが追加されました！")
```

これも問題なく動いて、データが追加されたっぽい！もう、僕はAIの回答を盲信し始めていました。「AIが言うことは絶対だ！」って。だって、僕が書きたいコードを、僕が知らない言語で、瞬時に、しかも完璧に書いてくれるんですから。これぞまさに「バイブコーディング」（フィーリングでコードを書くこと）の極みだと、当時は思っていましたね。

:::message
**補足：SQLiteについて**
SQLiteは、サーバーを必要とせず、ファイルとしてデータベースを保存する軽量なデータベースです。個人開発やちょっとしたツールにはとても便利ですが、複数の人が同時にアクセスするような大規模なシステムには向いていません。
:::

## まさかのデータ全消去！画面が真っ白になったあの夜

順調にデータを追加し、表示する機能もAIに教えてもらいながら実装を進めていました。ある夜、ふと「そういえば、データベースのデータを全部消したい時ってどうするんだろう？」って思ったんです。テストデータが溜まってきて、一度まっさらな状態にしたいな、と。

僕はいつものように、AIに尋ねました。
「SQLiteのデータベースに入っているデータを全て削除するPythonコードを教えてください。」

AIはすぐにコードを返してくれました。

```python:delete_all_data.py
import sqlite3

conn = sqlite3.connect('my_book_log.db')
cursor = conn.cursor()

# 全データを削除
cursor.execute("DELETE FROM books")

conn.commit()
conn.close()

print("データベースの全データが削除されました！")
```

「おお、これだ！」僕は深く考えずに、このコードをコピーして、実行しました。

画面には「データベースの全データが削除されました！」というメッセージが表示されました。
「よし！綺麗になった！」と、僕は満足げにデータ表示画面を開いたんです。

……しかし、そこに表示されたのは、真っ白な画面でした。

あれ？データが一つもない。もちろん、削除したんだから当たり前なんですけど、僕が想定していたのは、「特定のテーブルのデータだけが消える」ことでした。でも、この真っ白な画面は、まるでデータベースそのものが空っぽになったような、そんな絶望感を僕に与えました。

慌てて、以前追加したはずのデータがあるか確認するコードを走らせてみましたが、結果はやはり「0件」。僕がこれまでコツコツと入力してきた、架空の読書記録やテストデータ、そして本物の僕の読書記録……全てが消えていたんです。

深夜2時。画面の光だけが僕の顔を照らし、僕の心は真っ白になりました。「嘘だろ…？」って声が漏れました。あの時の絶望感は、今でも忘れられません。まさに、僕の『どん底体験』でした。

## なぜ僕のDBは消えたのか？AIとの対話の落とし穴

データが消えたことを理解するのに数分かかり、それからなぜこんなことになったのか必死に原因を探りました。まずは、AIに「今実行したコードで、なぜすべてのデータが消えたの？」と尋ねてみました。AIは丁寧に教えてくれました。

「`DELETE FROM books` というSQLコマンドは、`books` テーブル内の**すべてのレコード（行）**を削除します。特定の条件を指定しない限り、テーブル内のデータは全て削除されます。」

……なるほど。そういうことだったのか。

僕が欲しかったのは、「テストで入れたデータだけを削除する」機能でした。例えば、「idが5以下のデータだけ消す」とか、「タイトルが『テスト』って入ってるデータだけ消す」とか、そういう特定の条件でデータを消す方法だったんです。

でも、僕のプロンプトは「データベースに入っているデータを全て削除する」でした。AIは僕の言葉通りに、「`books` テーブルから全データを削除する」コードを生成したわけです。AIは僕の意図を汲み取ったのではなく、僕が入力した言葉をそのまま解釈しただけだったんです。

この時、僕はAIとの対話における大きな落とし穴に気づきました。

1.  **曖昧な指示の危険性**: 僕の「データを全て削除する」という指示は、AIにとっては「テーブル内の全レコードを削除する」と解釈されました。もし「テストデータのみを削除する」とか、「特定の条件に合致するデータだけを削除する」と具体的に指示していれば、結果は違っていたはずです。
2.  **AIの回答を盲信する危うさ**: AIが生成したコードは、一見すると完璧に見えます。しかし、そのコードが**何を意図しているのか、どんな影響があるのか**を、僕自身が理解していませんでした。AIが「正解」をくれると信じきっていたんです。
3.  **プログラミング知識の不足**: `DELETE FROM table_name` がテーブル内の全レコードを削除するSQLコマンドだという基本的な知識があれば、僕は実行前に立ち止まることができたはずです。AIを使うことで、基本的な知識の習得を怠ってしまったんですね。

:::message alert
**AI生成コードを実行する前の超重要ポイント！**
AIが生成したコードは、必ずしもあなたの意図を完璧に反映しているとは限りません。特に、データの削除や変更など、**元に戻せない操作**を行うコードを実行する前は、以下の点を確認しましょう。
*   コードが何をするものなのか、自分で理解する
*   影響範囲を把握する（例：全データ削除ではないか？）
*   可能であれば、バックアップを取るか、テスト環境で試す
:::

データが消えた原因がわかったところで、僕の絶望感は少し薄れ、代わりに「やっちまったな」という後悔と、大きな学びが生まれました。
